    About Me

    My name is Yi Wei Wu or also referred to as Jackson. I am a student in the Computer Science Specialist (Software Engineering Stream) program at University of Toronto Scarborough. I am currently on a 12-month work term as a QA analyst for Clearbridge Mobile Inc, which is a startup aimed at capturing all aspects of the creation of mobile applications. These aspects include design & analysis, code implementation, testing, and maintenance. Please enjoy the portfolio below of my personal work and inherited knowledge during my current work term.

    Portfolio

    1. Appium UI Automation Testing

    This artifact was produced on December 16, 2019 for the purpose of demonstrating achievements accomplished during the work term. In lieu of the actual projects due to confidentiality, this is a customly-tailored project to showcase similar aspects as covered in the actual projects.

    The goal of this artifact was to begin the investigation on UI automation testing. It was a task that was assigned to each of the interns so that we can familiarize ourselves with the tool. The artifact was developers for our supervisor to bring new knowledge if possible.

    When I first began on the task, I searched up guides on how to write test cases for automation, as well as tips and tricks for ui automation. This allowed me to quickly understand how Appium works hand-in-hand with the tested application. I was able to find this website that provided many threads on common problems (with their solutions) in Appium.

    The initial phase lasted for approximately two months, which was worked on on-and-off with a higher priority put on testing applications that were in development. At the end of this time period, each of the intern got to present their findings and their work to everyone else. This was proceeded by helpful criticism to further enhance the presenter's work.

    Reflecting on the time, I think there are a lot of areas I could've changed. For example in the beginning, I did not yet learn how to write proper test cases. This resulted in many hours of repetitive work. However, after learning how I can re-use the work I had done, I changed up the previous excessiveness and was able to make the project much more readable. This is something I will take and apply going forward on any future projects that will be candidates for UI automation testing.

    2. AngularJS Web Development

    This artifact was produced on April 4, 2020 for the purpose of demonstrating achievements accomplished during the work term. In lieu of the actual projects due to confidentiality, this is a customly-tailored project to showcase similar aspects as covered in the actual projects. The artifact that I have created here is a generalized version of my work. The "component" that I have created in this artifact is a form field. It can be re-used by introducing new form fields with different titles (depending on what the user names it). Artifact implementation can be found here.

    The goal of this artifact was to demonstrate my achievements in learning the web development language AngularJS. It was an extra task that I took on during this last work term in order to learn more about the field of software development. The artifact that I created is based off of the knowledge I gained during the ongoing progress of my assigned task, and it shows how AngularJS is able to create form-oriented webpages.

    When I first began on the task, I used the Udemy website provided by the company, which allowed me to take a 80-hour course on AngularJS. This was extremely helpful for my learning, because Udemy provides the user with the ability to pick and choose between lessons similar to UTSC's WebOption lectures. It helped me quickly understand how AngularJS works in its foundation. Furthermore, I was able to apply the skills directly through mini-projects provided at the end of each chapter!

    After I briefly learned about how AngularJS worked, I got to try out my new knowledge by building a "component" for the task. An AngularJS "component" in short is a building block that can be re-used over and over as a way to "build up" the project. Knowing that my "component" could be re-used, I had to plan out how I was going to develop it. This process included consulting with my supervisor on the task as well as researching online for common approaches to building "components".

    In the end, my planning helped me create a well-constructed "component" which I re-used many times to build more complex components. Reflecting on the time, I think I was able to learn from the issues I had during my first artifact. For example, instead of trying to figure it out as I go, I studied how the previous developer had done things and referred to online forums. This was helpful in guiding me to make good decisions during the process.
